# 2. 问题设计
智慧产业园区——多源事件收敛与约束派单调度系统设计
---
## 1. 业务背景

"星汇智慧产业园" 是某二线城市的一个在运营园区，占地约 18 万平方米，共 20 栋楼，入驻企业约 120 家，日均在园人数 8,000+。园区运营方希望引入一套智能体系统，接管当前以人工调度为主的运维指挥中心。

**运维资源**：

- 50 名一线人员：保洁 18、保安 12、设备维修 10、水电工 6、绿化 4，分两班倒
- 3 台巡检/保洁机器人（以下简称"机器人"），均为轮式，能自主导航，配备摄像头和简易清扫模块
- 1 名调度主管（目前靠微信群 + 对讲机调度）

**事件来源**（每日合计 5,000 条左右）：

1. **IoT 传感器告警**：电梯运行状态、水泵电流、空调水温、消防水压、地下车库 CO 浓度等，约 500+ 个监测点
2. **摄像头视觉识别**：园区 200+ 摄像头，AI 视觉每隔数秒推送识别结果，包括：污渍/遗留物/人员聚集/电动车违停/烟雾/明火
3. **住户/企业投诉**：来自三个渠道——园区 App 文本工单、物业微信群聊天记录（语音+文本+图片）、12345 市民热线转派工单
4. **巡检员现场报告**：保安和保洁在巡检时通过对讲机上报（语音转写）或 App 拍照上传

## 2. 附件

- `attachments/alerts_7days.csv` — 过去 7 天约 3.5 万条原始告警流（未去重、未分类）
- `attachments/complaints_sample.jsonl` — 过去一周约 350 条投诉样本（含文本/语音转写/图片文件名，有错别字和方言）
- `attachments/workforce.csv` — 50 名运维人员的技能、班次、常驻区域；3 台机器人的型号、能力
- `attachments/sla_matrix.md` — 运营方提供的事件类型 × SLA 要求对照表

# 4. 运行方式说明
- 执行黑盒跑批验证
运行验证脚本，系统将在一秒内吞吐 3.3 万条历史告警数据，并输出详尽的漏斗收敛报告：
```bash
python poc_main.py
```
- 启动可视化大屏
运行 UI 程序，直观观测 L1~L4 多级漏斗拦截与空间打包的动态过程：
```bash
python smart_monitor.py
```
