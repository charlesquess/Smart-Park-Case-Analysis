## 1. 优先级定义

| 级别 | 说明 |
|---|---|
| P0 | 紧急，影响生命安全 |
| P1 | 高，影响企业正常经营或涉及外部投诉 |
| P2 | 中等，一般性问题 |
| P3 | 低，可延后处理 |

## 3. 补充说明

1. "响应时限" 指从事件生成到执行人员到达现场；"处置时限" 指从到达现场到问题解决。两个定义是 2025-06 会议上口头约定的，尚未正式文档化。

2. 夜间（22:00 – 次日 07:00）非紧急事件可延后至次日上午处理。但 P0 事件不受此限。

3. 极端天气（台风、暴雨、冰冻）期间，所有事件响应时限统一延长 50%，P0 事件由周主管临时调度。

4. 节假日（春节、国庆）期间的特殊 SLA 由运营部临时下发，不在本表内。

5. 如遇本表未覆盖的事件类型，默认按 P2 处理，执行人员根据经验判断。

6. SLA 违约的处罚暂按内部 KPI 考核扣分，未形成书面制度。

## 4. 历史问题备注

- **2025-06**：某次业主投诉 "空调不制冷"，处置时限写 "次日"，实际发生在连续高温周导致客户严重不满，运营部被约谈。后续是否要按季节分级，讨论中。

- **2025-07**：安防类视觉识别事件（如人员聚集）如何与保安的日常巡逻工单合并，未形成规范。

- **2025-08**：12345 转派工单要求 "当日处置"，但 23:00 之后才派进来的案例按哪天算，内部尚无共识。

## 5. 工单状态要求

所有工单闭环后必须在物业管理系统填写：
- 到场时间
- 处置人签字
- 处置结果描述
- 照片（必要时）

12345 转派工单还需在 24 小时内通过市民热线回复接口反馈处理结果。
